## MultiStateNeuron

Cada neurônio possui:

* múltiplos estados
* memória por estado
* roteamento baseado na entrada

Exemplo:

```text
Estado 0
Estado 1
Estado 2
```

Uma entrada ativa apenas um estado por vez.

---

## V4Layer

A camada V4 contém vários neurônios.

Exemplo:

```text
V4Layer
├── Neuron 0
├── Neuron 1
├── Neuron 2
└── Neuron 3
```

Cada neurônio recebe uma versão da entrada e produz seu próprio estado.

## 1️⃣ Objetivo

Testar o comportamento do **MultiStateNeuron** com 5 estados para garantir:

## 2️⃣ Configuração do teste

- Número de estados: **5**
- Número de inputs: **1000**
- Método de roteamento: hash simples (`hash(str(x)) % states`)

Código de teste:

```python
from collections import Counter

class MultiStateNeuron:
    def __init__(self, states=5):
        self.states = states
        self.state_counter = Counter()

    def select_state(self, x):
        return hash(str(x)) % self.states

    def forward(self, x):
        state = self.select_state(x)
        self.state_counter[state] += 1
        return f"state_{state}_output"

    def report(self):
        total = sum(self.state_counter.values())
        print("\n=== RELATÓRIO DE ESTADOS ===")
        for state,count in sorted(self.state_counter.items()):
            pct = count / total * 100
            print(f"Estado {state}: {count} usos ({pct:.2f}%)")

# Teste
neuron = MultiStateNeuron(states=5)
for i in range(1000):
    neuron.forward(f"Tarefa {i}")

neuron.report()
````

# Próximo Passo (V5)

Agora você precisa criar competição entre estados.

Atualmente:

```python
estado = random.randint(0, 4)
```

ou alguma lógica equivalente.

Na V5 cada estado deve possuir um score:

```python
score = afinidade_categoria + experiencia
```

e o estado vencedor será:

```python
estado_escolhido = max(scores)
```

Isso é importante porque a especialização verdadeira só aparece quando os estados competem para resolver tarefas.

### Hipótese V5

Se os estados competirem por afinidade:

* Estados de código receberão mais tarefas de código.
* Estados matemáticos receberão mais tarefas matemáticas.
* Estados de planejamento receberão mais tarefas de planejamento.

Nesse ponto você começa a testar a principal hipótese da sua arquitetura:

> "Estados internos podem evoluir para especialistas independentes sem aumentar o número total de neurônios."

Esse é o primeiro experimento realmente interessante da linha MultiEstado.
